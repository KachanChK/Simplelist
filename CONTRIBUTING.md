# Guia de Contribuição

Em um projeto open-source, toda ajuda é bem-vinda, seja para corrigir um bug, sugerir uma nova funcionalidade ou melhorar a documentação.

## Como posso contribuir?

### Reportando Bugs
Se você encontrar algum erro ou comportamento inesperado:
1. Verifique se o erro já foi relatado nas nossas [Issues](https://github.com/KachanChK/Simplelist/issues).
2. Se não, abra uma nova issue descrevendo o erro, como reproduzi-lo e qual o comportamento esperado.

### Sugerindo Melhorias
Tem uma ideia ou apenas quer contribuir para o Simplelist?
1. Abra uma issue para discutirmos a funcionalidade ou escolha uma que esteja na lista de issues.
2. Isso ajuda a garantir que a mudança esteja alinhada com a proposta do projeto antes de você começar a contribuir.

### Enviando um Pull Request (PR)
1. Faça um **Fork** do projeto.
2. Crie um branch para sua modificação (`git checkout -b feature/nova-funcionalidade`).
3. Faça o **Commit** das suas alterações (`git commit -m 'feat: Add nova funcionalidade'`).
4. Faça o **Push** para o seu branch (`git push origin feature/nova-funcionalidade`).
5. Abra um **Pull Request**.

## Padrões de Código

- **Minimalismo:** O foco do Simplelist é ser simples. Evite adicionar complexidade desnecessária e tente manter a paleta de cores minimalista.
- **TypeScript:** Use tipos adequados para manter o padrão do código.
- **Estilização:** Priorize Tailwind CSS.
- **Commits:** Tente usar mensagens claras sobre o que foi feito (ex: `feat:`, `fix:`, `docs:`, `style:`).

---
Muito obrigado por colaborar! Keep it simple.
